B · The "BYO Figma" Product

Demo site + real Figma API integration. Hiring manager pastes their own Figma file URL. Ghost reads their actual design tokens (colors, type, spacing) via Figma's public REST API and compares against a hardcoded production sample. They see their own design system on the canvas. The "scan" output is curated but uses real data. This is when Ghost feels like a real product.

Wow: "Wait, it actually read my Figma file."
Effort: 6-8 weeks. A + Figma API plumbing + token visualization.
Defensibility: high. Real integration with a real platform.

Chrome extension. Scan ANY production page. Install Ghost. On any URL, click the extension. Highest virality. Hardest to demo cold (requires install).

Defensibility: highest — actually usable on real pages.

My recommendation: B · The BYO Figma Product

This is the sweet spot. A is polished but stays in "demo" territory — every hiring manager has seen impressive demos. C is the real-product move but requires a working scan engine that operates on arbitrary HTML, which is months of engine work that might never reach "this just works" quality on a portfolio timeline. D is the founder play but spreads polish across four surfaces; weak link breaks the whole story.

B threads the needle: real Figma API integration means hiring managers see their own tokens on Ghost's canvas, which is the precise moment "demo" becomes "product." The comparison side stays curated (hardcoded production samples per case), so we control quality. The scan output is realistic-looking, derived from real data on the design side, defensible end-to-end. 6-8 weeks of focused work, shippable, surprising. The portfolio reflection (Phase 2) then has a real product to point at, with a real Figma integration, hosted publicly.

If you have appetite for D, we can phase it: B becomes Phase 1, the browser extension comes in Phase 3 once B is shipped and tested in the wild. But starting with D is asking to ship nothing polished by month four.
