/ prompt · J K issues · 1-4 layers · ? help

1Today's priority strip answers the 3-second JTBD before the user touches anything. Animated entrance: types character-by-character first paint only, 800ms.

2Prompt is now secondary, a single line with / keyboard shortcut hint. The full chip row disappears (chips were a static feature-tour). Free-text prompt only.

3Page tabs carry severity signal via the leading dot color + issue count. Find the fire at a glance. Severity sorts left-to-right (worst first) on first paint; user can re-sort.

4Layer toggle with 4 distinct mini-previews. Each layer renders categorically different visuals: friction = thermal bloom, rage = sharp point-spikes with red rings, dead = grayscale voids with hatching, attention = horizontal gaze trail. Switching layers is a 400ms OKLab palette morph.

5Scrubber promoted above the canvas, with date labels and the delta of the current scrub position vs. start. Always visible, no scroll-to-find. Friction zone cards become first-class clickable rows with hover arrow + trend delta. Clicking flies the cursor to the issue in the canvas.

6"What changed this week" panel below the canvas — narrative auto-generated from the same data the Brief uses. Two columns (Degraded / Improved), each with 2 items max. Click any row → fly to that issue.

Bottom-right keyboard hint bar teaches shortcuts ambient. Auto-fades after first interaction or 4 seconds. ? opens a full shortcut help panel.

Not shown but planned: first-visit guided tour overlay (3 steps), last-session memory (page+layer+scrub position restored), light/dark toggle in left rail at the bottom. The Components / Brief drawers keep their current shape with the polish fixes (dynamic dates, mixed dataset).
